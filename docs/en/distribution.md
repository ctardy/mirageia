# Distribution and installation -- MirageIA

## Distribution strategy

### Principle

MirageIA is a **single binary** with no external dependency. Installation consists of:
1. Downloading the binary for your OS
2. Placing it in the PATH
3. Running it

No Docker, no Python, no Node, no runtime. A single executable file.

---

## Current phase -- Precompiled binary

### GitHub Releases

Each release publishes binaries for 3 platforms:

```
mirageia-v0.1.0-windows-x86_64.zip      (~5 MB)
mirageia-v0.1.0-macos-aarch64.tar.gz    (~5 MB)
mirageia-v0.1.0-linux-x86_64.tar.gz     (~5 MB)
```

Contents of each archive:
```
mirageia(.exe)       The binary
config.example.toml  Configuration example
README.md            Quick start instructions
```

### Manual installation

#### Windows

```powershell
# Download
Invoke-WebRequest -Uri "https://github.com/ctardy/mirageia/releases/latest/download/mirageia-windows-x86_64.zip" -OutFile mirageia.zip

# Extract
Expand-Archive mirageia.zip -DestinationPath "$env:LOCALAPPDATA\MirageIA"

# Add to PATH (PowerShell profile)
$env:PATH += ";$env:LOCALAPPDATA\MirageIA"
```

#### macOS / Linux

```bash
# Download and install
curl -sSfL https://github.com/ctardy/mirageia/releases/latest/download/mirageia-$(uname -s | tr '[:upper:]' '[:lower:]')-$(uname -m).tar.gz | tar xz -C /usr/local/bin/
```

### Digital signature (Windows)

The `mirageia.exe` binary is **digitally signed** with Microsoft Trusted Signing. The publisher displayed by Windows is `UITguard`.

This means:
- No SmartScreen warning on download or execution
- Integrity verification: any modification of the binary invalidates the signature
- RFC 3161 timestamp: the signature remains valid after the certificate expires

To verify the signature:
```powershell
Get-AuthenticodeSignature mirageia.exe
```

macOS and Linux binaries are not signed (out of scope for this phase).

### Automatic installation script

An `install.sh` script detects the OS and architecture, then downloads the correct binary:

```bash
curl -sSf https://raw.githubusercontent.com/ctardy/mirageia/main/install.sh | sh
```

The script:
1. Detects OS (Windows/macOS/Linux) and architecture (x86_64/aarch64)
2. Downloads the binary from GitHub Releases
3. Places it in `~/.local/bin/` (Linux/macOS) or `%LOCALAPPDATA%\MirageIA` (Windows)
4. Verifies the installation (`mirageia --version`)
5. Creates the `~/.mirageia/` directory if it does not exist

## First launch -- User experience

### Guided configuration: `mirageia setup`

The interactive wizard guides the developer through the entire configuration:

```bash
mirageia setup
```

The wizard:
1. **Detects the OS** (Windows/macOS/Linux) and shell (bash/zsh/PowerShell/Git Bash)
2. **Asks for the listening port** (default: 3100)
3. **Offers LLM providers** (Anthropic, OpenAI, Gemini, Mistral) -- auto-detects existing API keys
4. **Offers a whitelist** of terms that should never be pseudonymized
5. **Generates `~/.mirageia/config.toml`**
6. **Configures the shell** automatically (adds `export` statements to `.bashrc` / `.zshrc`)

On first launch without setup, MirageIA displays:
```
First time? Run `mirageia setup` for the guided configuration.
```

The proxy starts anyway with defaults -- setup is not mandatory.

## CI/CD -- Automated build

### GitHub Actions for releases

```yaml
# .github/workflows/release.yml
name: Release
on:
  push:
    tags: ['v*']

jobs:
  build:
    strategy:
      matrix:
        include:
          - os: windows-latest
            target: x86_64-pc-windows-msvc
            artifact: mirageia-windows-x86_64.zip
          - os: macos-latest
            target: aarch64-apple-darwin
            artifact: mirageia-macos-aarch64.tar.gz
          - os: ubuntu-latest
            target: x86_64-unknown-linux-gnu
            artifact: mirageia-linux-x86_64.tar.gz

    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
        with:
          targets: ${{ matrix.target }}
      - run: cargo build --release --target ${{ matrix.target }}
      - name: Package
        run: |
          # Create the archive with the binary + config.example.toml
          ...
      - uses: softprops/action-gh-release@v2
        with:
          files: ${{ matrix.artifact }}
```

## Updates

### Automatic updates (built-in)

MirageIA includes a transparent automatic update system:

```
Proxy startup
       │
       ├── 1. Check for a staged binary in ~/.mirageia/staging/
       │      → If found: swap with current binary, notify user
       │
       ├── 2. Start the proxy normally
       │
       └── 3. In the background (after 5s):
              → Check latest version on GitHub Releases
              → If newer: download to ~/.mirageia/staging/
              → Will be applied on next startup
```

**The user sees nothing** — on next restart, the new version is already there.

### Manual command

```bash
# Check if an update is available
mirageia update --check

# Check, download and apply immediately
mirageia update
```

### Swap mechanism

1. The new binary is downloaded to `~/.mirageia/staging/`
2. On startup, the current binary is renamed to `.old`
3. The staged binary is copied to the current location
4. `.old` and staging/ are cleaned up
5. On error: automatic rollback to `.old`

### Install script (first install or force latest version)

```bash
curl -sSf https://raw.githubusercontent.com/ctardy/mirageia/main/install.sh | sh
```

### cargo install

```bash
cargo install mirageia --force
```

## Binary sizes (estimates)

| Configuration | Estimated size |
|---|---|
| CLI only (without ONNX) | ~5-8 MB |
| CLI + ONNX Runtime | ~30-40 MB |
| Tauri (with webview, without ONNX) | ~15-20 MB |
| Tauri + ONNX | ~50-60 MB |
| PII model (DistilBERT INT8) | ~260 MB (downloaded separately) |

The ONNX model is downloaded on first launch into `~/.mirageia/models/`, not included in the binary.
